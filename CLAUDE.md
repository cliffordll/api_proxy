# CLAUDE.md

## 技术栈

- Python 3.11+ / FastAPI / httpx / pydantic-settings / pyyaml
- 测试: pytest + pytest-asyncio

## 编码规范

- 所有 I/O 使用 async/await
- 转换器（converters/）必须是纯函数，无 I/O，无副作用
- 流式响应使用 SSE 格式
- 配置统一通过 pydantic-settings，不直接 os.environ
- 不硬编码 API 密钥
- content 块使用多态类型设计，预留多模态扩展
- 模型映射从 config/model_mapping.yaml 加载，未命中则透传
- 做好封装，保持模块间职责清晰、接口稳定，便于后续扩展和替换

## 开发流程规范

开发任务必须通过 `docs/process.md` 进行管理，包含两部分：

**1. 任务计划表**：开始开发前，先将完整的任务计划输出到 `docs/process.md`，包含：
- 按 Phase 分组的任务列表（编号、任务、说明、产出文件、状态）
- 每个 Phase 的验收标准
- 任务状态标记（待开始/进行中/完成/阻塞）

**2. 执行记录**：每完成一个任务步骤，必须同步更新执行记录，包含：
- 当前完成的任务编号和内容
- 关键实现说明（设计决策、遇到的问题及解决方案）
- 测试数据和测试结果
- 阶段验收结论（通过/失败及原因）

**流程**：先创建完整计划 → 用户审核确认 → 严格按计划执行 → 逐步更新执行记录和任务状态

## 文档归档规范

docs/ 下的文档日常直接放在根目录使用。当用户发出归档指令（如"归档"、"存档"等）时：
1. 扫描 docs/ 下已有的 v* 目录，确定下一个版本号（从 v0.1 开始，依次 v0.2, v0.3...）
2. 在 docs/ 下创建对应版本目录（如 docs/v0.1/）
3. 将 docs/ 根目录下的文档文件移动到该版本目录中
4. 版本号自动管理，用户不需要指定
5. 归档后的文件只读不写，不得修改已归档版本目录中的任何文件
